## 커밋 컨벤션(예시)

팀 규칙이 없다면 아래처럼 가볍게 시작해도 좋습니다.

- **feat**: 기능 추가
- **fix**: 버그 수정
- **docs**: 문서 변경
- **refactor**: 리팩터링
- **chore**: 기타 작업(설정/의존성 등)

예: `docs: add project README`

## 기여 방법(예시)

1. 브랜치 생성: `git checkout -b feature/my-change`
2. 작업 후 커밋
3. 푸시: `git push origin feature/my-change`
4. PR 생성
